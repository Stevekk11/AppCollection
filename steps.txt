## Plán práce – krok za krokem
### 1. Základní funkce (nejdříve zprovoznit)
- [ ] Budíky: základní funkčnost – přidání, úprava, mazání (minimálně dva)
- [ ] Základní možnosti buzení (pípání, příroda, rádio)
- [ ] Změna času ručně + zobrazování aktuálního času v textové podobě („slovní hodiny“)
- [ ] Možnost změny barvy ciferníku
- [ ] Zobrazení data, teploty a základní předpovědi počasí

### 2. UI a základní struktura
- [ ] Navrhnout moderní a responsivní UI (Razor, CSS)
- [ ] Vytvořit základy pro více jazyků (lokalizace)

### 3. Rozšiřující funkce
- [ ] Pokročilá konfigurace budíků (opakování, snooze, nastavení na konkrétní den/hodinu)
- [ ] Možnost volby více zvuků a vlastního zvuku/barev
- [ ] Nastavení motivů (světlý/tmavý) + podpora vlastního obrázku
- [ ] Weather API – pokročilé údaje, více dnů, ikony, upozornění
- [ ] Uživatelské účty a správa uživatelských dat

### 4. Technické zázemí a kvalita
- [ ] Logging všech klíčových událostí a chyb
- [ ] Napsat sadu unit/integration testů pro funkce budíků, počasí, uživatelských účtů
- [ ] REST API pro seznam budíků atd.
- [ ] Export/záloha uživatelských dat

### 5. Dokumentace a dokončovací práce
- [ ] Vytvořit README a uživatelský manuál (česky i anglicky)
- [ ] Komentovat hlavní třídy a metody v kódu (XML komentáře)
- [ ] Otestovat, doladit a předat
